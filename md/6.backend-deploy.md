## 1. 아키텍처 개요

### 1.1 구성

| 계층 | 구성 요소 | 역할 |
|------|-----------|------|
| **노드** | `backend-ng` (t3.medium × 2) | 백엔드·RabbitMQ 전용 Worker Node |
| **Pod** | Spring Boot App Service | REST API, 비즈니스 로직, 큐 발행 |
| **Pod** | RabbitMQ | Frontend ↔ Backend ↔ AI FastAPI 비동기 메시지 중계 |
| **이미지** | ECR `forenshield-backend` | Spring Boot JAR 컨테이너 |
| **Service** | `backend` (ClusterIP, port 8080) | 클러스터 내부 API 트래픽 |
| **DB** | RDS PostgreSQL (db.t3.medium) | 사건·감사로그·분석결과 메타데이터 |
| **캐시** | ElastiCache Redis (cache.t3.medium) | 세션·작업 상태 캐시 |
| **스토리지** | S3 `forenshield-evidence` | 증거 원본 파일 (WORM) |

### 1.2 트래픽 · 연동 흐름

```text
Frontend Pod
    → HTTP 8080 → Backend Pod
                    → 5432 → RDS PostgreSQL
                    → 6379 → ElastiCache Redis
                    → 5672 → RabbitMQ Pod
                                → 큐 메시지 → AI FastAPI Pod
                                                → GPU 추론 → On-Prem GPU Gateway
                    → IRSA  → S3 forenshield-evidence
ALB / Ingress
    → /api/* → Backend Pod
    → /*     → Frontend Pod
```

### 1.3 배포 순서 (전체 파이프라인 내 위치)

백엔드는 RabbitMQ · AI FastAPI 선행 배포 후, 프론트엔드 배포 전에 진행합니다.

```text
Namespace
 └─ Secrets / ConfigMap
      └─ RabbitMQ              ← 선행 필수
           └─ AI FastAPI        ← 큐 소비자 (선행 권장)
                └─ Backend      ← 본 가이드
                     └─ Frontend
                          └─ Ingress / ALB
```

> RabbitMQ 없이 Backend만 올리면 큐 기반 분석 요청 API가 동작하지 않습니다. DB · Redis 연결만 먼저 검증하려면 RabbitMQ auto-config를 비활성화한 프로파일로 임시 기동할 수 있습니다.  
> RabbitMQ · AI FastAPI 배포 절차는 [AI 배포 가이드](./7.ai-deploy.md)를, RDS · Redis · S3 구축은 [데이터 레이어 배포 가이드](./4.data-layer-deploy.md)를 참고하세요.

## 3. 환경변수 · Secret 관리

환경변수 분류·매핑 규격의 총괄 정책은 [환경변수 설정 가이드](./3.settings.md)를 참고하세요.

### 3.1 변수 분류

| 구분 | 저장 위치 | 예시 | 민감 |
|------|-----------|------|------|
| DB 접속 정보 | Secret `db-credentials` | `POSTGRES_HOST`, `POSTGRES_USER`, `POSTGRES_PASSWORD` | Y |
| Redis 접속 정보 | Secret `redis-credentials` | `REDIS_HOST`, `REDIS_PASSWORD` | Y |
| RabbitMQ 인증 | Secret `rabbitmq-credentials` | `RABBITMQ_USER`, `RABBITMQ_PASSWORD` | Y |
| S3 · AWS | Secret `s3-config` + IRSA | `S3_EVIDENCE_BUCKET`, `AWS_REGION` | N |
| 앱 설정 | ConfigMap `backend-config` | `SPRING_PROFILES_ACTIVE`, `SERVER_PORT` | N |
| RabbitMQ 호스트 | ConfigMap `backend-config` | `RABBITMQ_HOST` | N |

### 3.2 Spring Boot 환경변수 매핑

| 환경변수 (K8s) | Spring 프로퍼티 | 설명 |
|----------------|-----------------|------|
| `POSTGRES_HOST` | `spring.datasource.url` (조합) | RDS 엔드포인트 |
| `POSTGRES_USER` | `spring.datasource.username` | DB 사용자 |
| `POSTGRES_PASSWORD` | `spring.datasource.password` | DB 비밀번호 |
| `POSTGRES_DB` | `spring.datasource.url` (조합) | DB 이름 (`forenshield`) |
| `REDIS_HOST` | `spring.data.redis.host` | Redis 엔드포인트 |
| `REDIS_PASSWORD` | `spring.data.redis.password` | Redis AUTH |
| `RABBITMQ_HOST` | `spring.rabbitmq.host` | K8s Service DNS |
| `RABBITMQ_USER` | `spring.rabbitmq.username` | RabbitMQ 사용자 |
| `RABBITMQ_PASSWORD` | `spring.rabbitmq.password` | RabbitMQ 비밀번호 |
| `S3_EVIDENCE_BUCKET` | `aws.s3.evidence-bucket` | 증거 버킷명 |
| `AWS_REGION` | `aws.region` | `ap-northeast-2` |
| `SPRING_PROFILES_ACTIVE` | — | `prod` |

#### application-prod.yml 예시

```yaml
spring:
  datasource:
    url: jdbc:postgresql://${POSTGRES_HOST}:5432/${POSTGRES_DB:forenshield}
    username: ${POSTGRES_USER}
    password: ${POSTGRES_PASSWORD}
  jpa:
    hibernate:
      ddl-auto: validate   # 운영: validate 사용. update 절대 금지
  data:
    redis:
      host: ${REDIS_HOST}
      port: 6379
      password: ${REDIS_PASSWORD}
  rabbitmq:
    host: ${RABBITMQ_HOST}
    port: 5672
    username: ${RABBITMQ_USER}
    password: ${RABBITMQ_PASSWORD}
aws:
  region: ${AWS_REGION:ap-northeast-2}
  s3:
    evidence-bucket: ${S3_EVIDENCE_BUCKET:forenshield-evidence}
management:
  endpoints:
    web:
      exposure:
        include: health,info
  endpoint:
    health:
      show-details: when_authorized
```

### 3.3 Secret 생성

터미널 히스토리에 비밀번호가 남지 않도록 `--from-env-file` 방식을 사용합니다.

각 Secret용 env 파일을 먼저 작성합니다. (Git 커밋 금지 — `.gitignore`에 `secrets/*.env` 추가)

```bash
# secrets/db.env
POSTGRES_HOST=<RDS_ENDPOINT>
POSTGRES_USER=forenshield
POSTGRES_PASSWORD=<PASSWORD>
POSTGRES_DB=forenshield

# secrets/redis.env
REDIS_HOST=<REDIS_ENDPOINT>
REDIS_PASSWORD=<PASSWORD>

# secrets/rabbitmq.env
RABBITMQ_USER=forenshield
RABBITMQ_PASSWORD=<PASSWORD>

# secrets/s3.env
S3_EVIDENCE_BUCKET=forenshield-evidence
AWS_REGION=ap-northeast-2
```

#### Secret 적용

```bash
kubectl create secret generic db-credentials -n forenshield \
  --from-env-file=secrets/db.env
kubectl create secret generic redis-credentials -n forenshield \
  --from-env-file=secrets/redis.env
kubectl create secret generic rabbitmq-credentials -n forenshield \
  --from-env-file=secrets/rabbitmq.env
kubectl create secret generic s3-config -n forenshield \
  --from-env-file=secrets/s3.env
```

> Secret 값은 Git에 커밋하지 않습니다. 장기적으로 AWS Secrets Manager + External Secrets Operator 연동을 권장합니다.  
> `.env.example` 규격 및 `.gitignore` 검증은 [환경변수 설정 가이드](./3.settings.md)를 참고하세요.

### 3.4 IRSA (S3 접근)

Access Key 없이 Pod ServiceAccount에 IAM Role을 연결하는 방식입니다.

#### IAM Trust Policy (EKS OIDC)

```json
{
  "Effect": "Allow",
  "Principal": {
    "Federated": "arn:aws:iam::<AWS_ACCOUNT_ID>:oidc-provider/oidc.eks.ap-northeast-2.amazonaws.com/id/<OIDC_ID>"
  },
  "Action": "sts:AssumeRoleWithWebIdentity",
  "Condition": {
    "StringEquals": {
      "oidc.eks.ap-northeast-2.amazonaws.com/id/<OIDC_ID>:sub": "system:serviceaccount:forenshield:backend-sa",
      "oidc.eks.ap-northeast-2.amazonaws.com/id/<OIDC_ID>:aud": "sts.amazonaws.com"
    }
  }
}
```

#### IAM Policy (S3 evidence 버킷 최소 권한)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::forenshield-evidence",
        "arn:aws:s3:::forenshield-evidence/*"
      ]
    }
  ]
}
```

## 4. Docker 이미지 빌드

### 4.1 Dockerfile (Gradle 멀티스테이지)

```dockerfile
# ---- Stage 1: 빌드 ----
FROM eclipse-temurin:17-jdk-alpine AS builder
WORKDIR /app
COPY gradlew .
COPY gradle gradle
COPY build.gradle settings.gradle ./
RUN ./gradlew dependencies --no-daemon
COPY src src
RUN ./gradlew bootJar --no-daemon -x test

# ---- Stage 2: 실행 ----
FROM eclipse-temurin:17-jre-alpine AS runner
WORKDIR /app
RUN addgroup -S spring && adduser -S spring -G spring
USER spring:spring
COPY --from=builder /app/build/libs/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", \
  "-XX:+UseContainerSupport", \
  "-XX:MaxRAMPercentage=75.0", \
  "-Djava.security.egd=file:/dev/./urandom", \
  "-jar", "app.jar"]
```

Maven 프로젝트는 `mvn package` + `target/*.jar` 경로로 교체합니다.

### 4.2 로컬 빌드 · 실행 테스트

```bash
cd <backend-repo>

# JAR 빌드
./gradlew bootJar -x test

# Docker 빌드
docker build -t forenshield-backend:local .

# 로컬 실행
docker run -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=local \
  -e POSTGRES_HOST=host.docker.internal \
  -e POSTGRES_USER=forenshield \
  -e POSTGRES_PASSWORD=<PASSWORD> \
  -e POSTGRES_DB=forenshield \
  -e REDIS_HOST=host.docker.internal \
  -e REDIS_PASSWORD=<PASSWORD> \
  -e RABBITMQ_HOST=host.docker.internal \
  -e RABBITMQ_USER=forenshield \
  -e RABBITMQ_PASSWORD=<PASSWORD> \
  forenshield-backend:local

# 헬스체크
curl http://localhost:8080/actuator/health
```

## 5. ECR Push

### 5.1 변수 설정

```bash
export AWS_REGION=ap-northeast-2
export AWS_ACCOUNT_ID=<12자리 계정 ID>
export IMAGE_TAG=$(git rev-parse --short HEAD)   # commit SHA 사용
export ECR_REPO=$AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/forenshield-backend
```

### 5.2 로그인 · 빌드 · Push

```bash
# ECR 로그인
aws ecr get-login-password --region $AWS_REGION \
  | docker login --username AWS --password-stdin \
    $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com

# 빌드
docker build -t forenshield-backend:$IMAGE_TAG .

# 태그 · Push
docker tag forenshield-backend:$IMAGE_TAG $ECR_REPO:$IMAGE_TAG
docker tag forenshield-backend:$IMAGE_TAG $ECR_REPO:latest
docker push $ECR_REPO:$IMAGE_TAG
docker push $ECR_REPO:latest
```

### 5.3 Push 확인

```bash
aws ecr describe-images \
  --repository-name forenshield-backend \
  --image-ids imageTag=$IMAGE_TAG
```

### 6.3 Deployment

`k8s/backend/deployment.yaml`

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: backend
  namespace: forenshield
  labels:
    app: backend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: backend
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: backend
    spec:
      serviceAccountName: backend-sa
      nodeSelector:
        nodegroup: backend-ng
      containers:
        - name: backend
          image: <ECR_REGISTRY>/forenshield-backend:<IMAGE_TAG>
          # CI/CD에서 kubectl set image로 교체. 직접 수정 금지
          imagePullPolicy: Always
          ports:
            - name: http
              containerPort: 8080
              protocol: TCP
          envFrom:
            - configMapRef:
                name: backend-config
            - secretRef:
                name: db-credentials
            - secretRef:
                name: redis-credentials
            - secretRef:
                name: rabbitmq-credentials
            - secretRef:
                name: s3-config
          readinessProbe:
            httpGet:
              path: /actuator/health
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 10
            failureThreshold: 3
          livenessProbe:
            httpGet:
              path: /actuator/health
              port: 8080
            initialDelaySeconds: 60
            periodSeconds: 15
            failureThreshold: 3
          resources:
            requests:
              cpu: 250m
              memory: 512Mi
            limits:
              cpu: 1000m
              memory: 1Gi
```

        - protocol: TCP
          port: 5432
    - to:                          # RDS (Data Subnet AZ-b)

            cidr: 10.0.21.0/24
      ports:
        - protocol: TCP
          port: 5432

            cidr: 10.0.20.0/24

### 6.6 DB 마이그레이션

스키마 변경은 앱 기동 전 Init Container로 실행합니다.

```yaml
initContainers:
  - name: flyway-migrate
    image: <ECR_REPO>:<IMAGE_TAG>
    command: ["java", "-jar", "app.jar", "--spring.profiles.active=prod,migrate"]
    envFrom:
      - secretRef:
          name: db-credentials
```

Flyway / Liquibase 사용 여부는 프로젝트 설정에 맞게 조정합니다. `ddl-auto: update`는 운영 환경에서 절대 사용하지 않습니다.

## 7. 외부 API 노출 (Ingress)

백엔드는 ClusterIP로 기본 배포하며, 외부 노출은 ALB Ingress의 `/api` 경로로 프론트와 분리합니다.

### 7.1 Ingress 경로 분리

`k8s/ingress/ingress.yaml`

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: forenshield-ingress
  namespace: forenshield
  annotations:
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTPS":443}]'
    alb.ingress.kubernetes.io/certificate-arn: <ACM_CERT_ARN>
    alb.ingress.kubernetes.io/ssl-redirect: "443"
    alb.ingress.kubernetes.io/healthcheck-path: /actuator/health
spec:
  ingressClassName: alb
  rules:
    - host: <your-domain>
      http:
        paths:
          - path: /api
            pathType: Prefix
            backend:
              service:
                name: backend
                port:
                  number: 8080
          - path: /
            pathType: Prefix
            backend:
              service:
                name: frontend
                port:
                  number: 80
```

> `spring.boot.server.servlet.context-path=/api` 설정 시 Ingress path와 반드시 일치시켜야 합니다.

### 7.2 Security Group 확인

| SG | 규칙 |
|----|------|
| `sg-alb` | 443 from `0.0.0.0/0` |
| `sg-eks-node` | 8080 from ALB SG |
| `sg-rds` | 5432 from EKS Node SG |
| `sg-redis` | 6379 from EKS Node SG |

## 9. CI/CD 파이프라인

### 9.1 파이프라인 흐름

```text
GitHub Push (main)
  → GitHub Actions: test (unit + integration)
  → Gradle bootJar + Docker build
  → ECR Push (commit SHA 태그 + latest)
  → kubectl set image
  → Rolling Update
  → /actuator/health 검증
```

### 9.2 GitHub Actions Secrets

| Secret | 설명 | 민감 |
|--------|------|------|
| `AWS_ROLE_ARN` | OIDC AssumeRole 대상 ARN | Y |
| `AWS_REGION` | `ap-northeast-2` | N |
| `ECR_REGISTRY` | `<account>.dkr.ecr.ap-northeast-2.amazonaws.com` | Y |
| `EKS_CLUSTER_NAME` | `forenshield` | N |

> DB · Redis 비밀번호는 CI Secret에 넣지 않습니다. K8s Secret은 별도 관리(수동 1회 apply)합니다.

### 9.3 GitHub Actions 워크플로

`.github/workflows/backend-deploy.yml`

```yaml
name: Backend Deploy
on:
  push:
    branches: [main]
    paths:
      - "backend/**"
      - ".github/workflows/backend-deploy.yml"
permissions:
  id-token: write
  contents: read
env:
  AWS_REGION: ap-northeast-2
  ECR_REPOSITORY: forenshield-backend
  EKS_CLUSTER: forenshield
  K8S_NAMESPACE: forenshield
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up JDK 17
        uses: actions/setup-java@v4
        with:
          java-version: "17"
          distribution: "temurin"
          cache: gradle
      - name: Run tests
        run: ./gradlew test
        working-directory: backend
      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}
      - name: Login to ECR
        id: ecr
        uses: aws-actions/amazon-ecr-login@v2
      - name: Build and push
        env:
          IMAGE_TAG: ${{ github.sha }}
          ECR_REGISTRY: ${{ steps.ecr.outputs.registry }}
        run: |
          docker build \
            -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG \
            -t $ECR_REGISTRY/$ECR_REPOSITORY:latest \
            ./backend
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:latest
      - name: Update kubeconfig
        run: aws eks update-kubeconfig --name $EKS_CLUSTER --region $AWS_REGION
      - name: Deploy to EKS
        env:
          IMAGE_TAG: ${{ github.sha }}
          ECR_REGISTRY: ${{ steps.ecr.outputs.registry }}
        run: |
          kubectl set image deployment/backend \
            backend=$ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG \
            -n $K8S_NAMESPACE
          kubectl rollout status deployment/backend \
            -n $K8S_NAMESPACE --timeout=300s
      - name: Health check
        run: |
          kubectl run health-check --rm -i --restart=Never \
            --image=curlimages/curl -n $K8S_NAMESPACE -- \
            curl -sf http://backend:8080/actuator/health
```

## 11. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| ImagePullBackOff | ECR 태그 오류 · 노드 IAM 권한 | 태그 확인, Node Role에 `AmazonEC2ContainerRegistryReadOnly` 부여 |
| Pod CrashLoopBackOff | DB / Redis 연결 실패 | Secret 값 · SG(5432/6379) 확인 |
| Connection refused (RabbitMQ) | RabbitMQ 미배포 · 호스트 오류 | `RABBITMQ_HOST` Service DNS 확인 |
| `/actuator/health` DOWN (DB) | RDS SG 또는 자격증명 오류 | EKS Node SG → RDS 5432 허용 확인 |
| S3 AccessDenied | IRSA Role / Policy 오류 | ServiceAccount annotation · IAM Policy 확인 |
| API 502 (Ingress) | Readiness Probe 미통과 | `initialDelaySeconds` 증가 (Spring 기동 30~60s) |
| Flyway migration 실패 | 스키마 충돌 | 마이그레이션 로그 확인 후 수동 repair |
| OOMKilled | JVM heap 과다 | `-XX:MaxRAMPercentage=75`, limits 1Gi 조정 |
| 큐 메시지 미전달 | Exchange / Queue 바인딩 오류 | RabbitMQ Management UI 또는 `rabbitmqctl` 확인 |

### 로그 확인
