## 개요

### 하는 일

| 동작 | 방법 | 소요 |
|------|------|------|
| EKS 워커 EC2 끄기 | Node Group `desiredSize=0` | 2~5분 |
| RDS 끄기 | `stop-db-instance` | 1~3분 |
| EKS 워커 켜기 | Node Group `desiredSize` 복구 | 5~10분 |
| RDS 켜기 | `start-db-instance` + wait | 5~15분 |

### 데이터 보존 (삭제하지 않음)

| 리소스 | 퇴근 후 | 데이터 |
|--------|---------|--------|
| S3 `forenshield-evidence-*`, `forenshield-models-*` | 그대로 | ✅ 유지 |
| RDS `forenshield-db` | **stopped** | ✅ 유지 |
| ECR 이미지 | 그대로 | ✅ 유지 |
| EKS 클러스터 · Node Group 설정 | 그대로 | ✅ 유지 |
| K8s 매니페스트 / ArgoCD | 그대로 (Pod만 없음) | ✅ 유지 |

### 끄지 않는 것 (고정 비용)

| 리소스 | 이유 |
|--------|------|
| EKS 컨트롤 플레인 | Stop API 없음 (~$73/월) |
| NAT Gateway | 삭제 시 복구 번거로움 (~$32+/월) |
| ALB | 노드 0이면 트래픽 없음, 시간당 요금 일부 잔존 |
| ElastiCache Redis | **Stop API 없음** — 당분간 유지 (캐시만, RDS와 별개) |

## 퇴근 시 — 중지 (Shutdown)

> **순서:** EKS 노드 먼저 → RDS (Pod가 DB 연결 끊긴 뒤 중지)

### 1) EKS Node Group 스케일 0

**PowerShell:**

```powershell
foreach ($ng in @("frontend-ng", "backend-ng", "ai-fastapi-ng")) {
  Write-Host "Scaling $ng to 0..."
  aws eks update-nodegroup-config `
    --cluster-name $env:CLUSTER_NAME `
    --nodegroup-name $ng `
    --scaling-config minSize=0,desiredSize=0,maxSize=4
}
```

**Git Bash (한 줄씩):**

```bash
for ng in frontend-ng backend-ng ai-fastapi-ng; do
  echo "Scaling $ng to 0..."
  aws eks update-nodegroup-config \
    --cluster-name "$CLUSTER_NAME" \
    --nodegroup-name "$ng" \
    --scaling-config minSize=0,desiredSize=0,maxSize=4
done
```

2~5분 후 EC2가 `terminated` 됩니다.

### 2) RDS 중지

```powershell
aws rds stop-db-instance --db-instance-identifier forenshield-db
```

이미 stopped면 `InvalidDBInstanceState` — 무시해도 됩니다.

### 3) 중지 확인

```powershell
# EKS 노드 0대
aws ec2 describe-instances `
  --filters "Name=tag:eks:cluster-name,Values=forenshield" "Name=instance-state-name,Values=running" `
  --query "Reservations[*].Instances[*].InstanceId" `
  --output text

# RDS stopped
aws rds describe-db-instances --db-instance-identifier forenshield-db `
  --query "DBInstances[0].DBInstanceStatus" --output text
```

기대값: EC2 출력 **없음**, RDS **`stopped`**

- `https://forensheildjangdochi.com` — 접속 불가 (정상)

## 출근 시 — 시작 (Startup)

> **순서:** RDS 먼저 → EKS 노드 → Pod 확인  
> RDS가 `available` 되기 전에 백엔드 Pod가 뜨면 DB 연결 오류가 날 수 있습니다.

### 1) RDS 시작

```powershell
aws rds start-db-instance --db-instance-identifier forenshield-db

aws rds wait db-instance-available --db-instance-identifier forenshield-db
```

5~15분 걸릴 수 있습니다. `available` 될 때까지 기다립니다.

### 2) EKS Node Group 복구

```powershell
aws eks update-nodegroup-config --cluster-name forenshield --nodegroup-name frontend-ng   --scaling-config minSize=1,desiredSize=1,maxSize=2
aws eks update-nodegroup-config --cluster-name forenshield --nodegroup-name backend-ng    --scaling-config minSize=1,desiredSize=2,maxSize=4
aws eks update-nodegroup-config --cluster-name forenshield --nodegroup-name ai-fastapi-ng --scaling-config minSize=1,desiredSize=1,maxSize=2
```

### 3) kubeconfig · 노드 Ready 대기

```powershell
aws eks update-kubeconfig --name forenshield --region ap-northeast-2

kubectl get nodes -w
```

`Ready` 4대 나오면 `Ctrl+C`.

### 4) Pod 기동 확인

```powershell
kubectl get pods -n forenshield
kubectl get pods -n argocd
```

| Pod | 기대 |
|-----|------|
| `frontend`, `backend`, `ai-fastapi`, `rabbitmq` | `Running` |
| ArgoCD | `Running` |

5~10분 지나도 `Pending`이면 `kubectl describe pod <name> -n forenshield` 로 이벤트 확인.

### 5) 서비스 헬스체크

```powershell
curl.exe -I https://forensheildjangdochi.com/
curl.exe https://forensheildjangdochi.com/actuator/health
```

## 비용 · 주의사항

### 절감되는 비용

| 리소스 | 중지 시 |
|--------|---------|
| EKS 노드 t3.medium ×4 | EC2 시간 과금 **0** |
| RDS db.t3.medium | 인스턴스 시간 과금 **0** |

### 절대 하지 말 것

| 명령 | 결과 |
|------|------|
| `aws rds delete-db-instance` | **DB 데이터 삭제** |
| `aws eks delete-cluster` | 클러스터 전체 삭제, 재구축 필요 |
| `aws s3 rb --force` | S3 버킷·증거 파일 삭제 |
| `aws eks delete-nodegroup` | 스케일 0으로 충분, 삭제는 복구 단계 증가 |

### RDS Stop 제한

- Stop 상태는 **최대 7일** — 이후 AWS가 자동으로 다시 시작할 수 있음 (데이터는 유지)
- 7일 이상 휴식이면 팀과 장기 정책 별도 논의

### Redis (ElastiCache)

- Stop API 없음 — 일일 루틴에서는 **그대로 둠**
- Redis 데이터(세션 캐시)는 날아가도 서비스 재기동은 가능하나, 당일 빠른 복구를 위해 유지 권장

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `InvalidClientTokenId` | `forenshield` 프로필 미설정 | [공통 설정](#공통-설정-매-터미널-1회) |
| `minSize cannot be greater than desiredSize` | min이 desired보다 큼 | `minSize=0,desiredSize=0` 같이 지정 |
| Pod `Pending` / `Insufficient cpu` | 노드 아직 부족 | `kubectl get nodes` Ready 대기 |
| Backend `CrashLoopBackOff` | RDS 아직 stopped | RDS `available` 후 Pod 재시작 |
| `kubectl` 연결 실패 | kubeconfig 만료 | `aws eks update-kubeconfig --name forenshield` |
| 사이트 안 열림 | ALB 타깃 unhealthy | Pod Running 후 2~3분 더 대기 |

Backend Pod 재시작 (RDS 올린 뒤):

```powershell
kubectl rollout restart deployment/backend -n forenshield
kubectl rollout restart deployment/frontend -n forenshield
```
