### 2.2 기술 스택·흐름 (팀 설명)

```text
사용자 → ALB/Route53/HTTPS → Frontend / Backend (EKS)
       → S3 evidence original 저장 + SHA-256
       → RabbitMQ → AI FastAPI Pod → GPU Gateway (HTTP, welabs RTX 5080)
       → RDS / Redis → PDF·JSON 리포트
```

- **S3:** `original/` 원본, GPU는 **original에서 Get → `/tmp` 작업 복사본 → 분석 후 삭제**
- **모델 버킷:** S3 `forenshield-models` 버전 관리 (배포 시 sync)

## 3. 멘토 핵심 피드백 (오전) — 반드시 숙지

### 3.1 “체인·해시·서명” 질문에 답 못 함 → 치명적

| 팀 설명 (당시) | 멘토 지적 |
|----------------|-----------|
| SHA-256 생성 | 해시 **생성만**으로는 “원본이 정확하다”는 **판별 근거가 될 수 없음** |
| “해시 체인으로 감사 로그 무결성” | **체인(Chain)이 정확히 무엇**인지, **전자서명** 들어가는지 **블록체인**인지 구분 없음 |
| “자동 무결성 인증 체인” | 마케팅 문구 수준 — **프로세스·표준·벤치마크** 없음 |

**멘토가 요구한 답변 구조:**

1. **업로드** → SHA-256 → **디지털 서명** → Ledger(CoC 해시 체인) → (선택) 블록체인 타임스탬프  
2. **나중에 “조작됐다”** 주장 시 → **검증 API/프로세스**로 반박 (말로 X)  
3. **C2PA** 등 **국제 표준** 조사·벤치마킹 — “우리만의 카피”가 아닌 **차별점** 명시  
4. **S3만**으로 부족하면 **보완**이 무엇인지 (서명, TSA, Object Lock) **역할 분리**해서 설명

### 3.2 C2PA (Content Credentials)

- **C2PA:** 디지털 미디어 **생성·수정 이력·출처** 기록·검증 **개방형 국제 표준**
- 가짜 뉴스·AI 생성 콘텐츠 식별 — **팀 주제와 직접 관련**
- **과제:** 표준 읽기 · 국내 도입 사례 · **ForenShield 차별점** (C2PA + CoC + 블록체인 앵커 등) 문서화

### 3.3 AI Ops / 모델 거버넌스

| 이슈 | 멘토 요구 |
|------|-----------|
| “모델 3개 테스트했다” | **6개월 후에도 같은 모델?** → **버전·프롬프트·A/B** 관리 |
| S3 모델 sync | **새 모델 = 무조건 좋음 X** → **골든셋**으로 기존 대비 개선 검증 |
| CI/CD CD | **Critical 영역(포렌식)** → **Human-in-the-loop** 배포 (완전 자동 CD 지양) |
| 용어 | DevSecOps + **AI Ops** 프로세스 문서화 (모델 선정 → 배포까지) |

### 3.4 MVP·범위·시간

- **완벽한 포렌식 X → MVP** (발표 ~2주, 실개발 ~1주+)
- **“나중에 하겠습니다” 금지** — 지금 하기로 한 것만 집중
- **음성 only 확대 X** — **영상·이미지·무결성 플로우** 우선 (시간 남으면 음성)
- AI가 코드·문서 만들어준 것 **≠ 팀 소유 지식** — **논문·표준·벤치마크** 기반 이해 필수

### 3.5 UI/기획 피드백

- 메인·상세·관리자 **화면은 있으나** “엔진별 가중치·사법적 타임라인” **예시/미구현** 혼재
- **차별화 3번**(해시 불일치 시 사용자 경고) → **차별점으로 부적절**, 수정
- **CoC 타임라인** = 사용자용 요약 + **관리자 상세 추적** 분리 필요

## 4. 오후 세션 — 인프라·저장소

### 4.1 VPN (Site-to-Site) vs GPU 공인 IP

| 항목 | 내용 |
|------|------|
| **기존** | VPN(IPsec)으로 EKS ↔ GPU, **2조와 터널 충돌·1h 키 리프레시** 이슈 |
| **현재** | VPN **삭제**, GPU **공인 IP + 포트** + AWS Access Key on GPU |
| **2조** | 이상탐지(CCTV) — **GPU/키 공유 아님**, AWS IAM 사용자 **분리** |
| **멘토** | VPN **다시 쓸 수 있으면 쓰는 게 좋음** (보안); 1h rekey는 **공존 가능**했음 |
| **결정(팀)** | VPN 재도입 **검토** — Customer Gateway / VGW / IPsec, GPU에 **AWS 키 직접 넣지 않는** 방향 |

> 상세 콘솔 절차: [`2.aws-architecture.md`](./2.aws-architecture.md) Step 9

### 4.2 S3 vs 블록체인 — 역할 재정의

**팀 초기 혼선:** “블록체인이 S3 **대체**” vs “S3 Object Lock이면 블록체인 불필요”

**멘토·팀 합의 (정리):**

| 레이어 | 역할 | 선택 |
|--------|------|------|
| **대용량 파일 (GB)** | 영상·음성·이미지 **바이트** 보관 | **S3 필수** (블록체인에 MP4 통째 X) |
| **변조 방지 (저장소)** | 삭제·덮어쓰기 어렵게 | S3 **Object Lock (Compliance WORM)** + 버전 관리 |
| **해시·감사·시각** | Ledger, 외부 증명 | **CoC 해시 체인(DB)** + (로드맵) **블록체인/TSA에 해시만** |
| **블록체인** | “저장소 교체” ❌ → **무결성·타임스탬프 보조** ⭕ |

**S3를 쓰는 이유 (발표용):**

- GPU·EKS·멀티모달 **실파일** 파이프라인
- **Managed** Object Lock·버전·IRSA·VPC Endpoint
- 블록체인은 **64자 해시·메타** 수준 — **증거 파일 창고 아님**

**부캠프 맥락:**

- AWS S3 **종속 자체는 문제 아님** (Managed 활용)
- 다만 **“정부·수사기관” 시나리오** 확장 시 **국내 클라우드(NCP·KT)** 이슈 언급 — **ForenShield MVP는 AWS 유지**, 시나리오만 aware

### 4.3 Object Lock (팀 설명)

- **Write Once Read Many (WORM)**, Compliance 모드, **365일** 등 보존
- **버전 관리:** 덮어쓰기 시 이전 버전 보존
- **→ 블록체인 “저장소” 대신 S3 WORM + (선택) on-chain hash** 로 기획 정렬

## 5. 서비스 정의 — 오후 확정 (가장 중요)

### 5.1 두 가지 핵심 기능 (팀·멘토 합의)

| # | 기능 | 설명 | 출력 |
|---|------|------|------|
| **①** | **Deepfake / AI 합성 탐지** | 올린 자료가 **AI·합성 의심**인지 (법적 “문제” 판단 X, **정보 제공**) | 구간별 의심 점수, XAI(히트맵·타임라인·스펙트rogram 등) |
| **②** | **위·변조(편집) 검증** | **이미 등록·서명된 원본** vs **새로 제출된 파일** 비교 — **변조 여부** | diff·구간별 편집 의심, 무결성 검증 결과 |

> **명확화:** “딥페이크라서 불법” ❌ → “**딥페이크·합성 패턴이 의심됩니다**” ⭕ (경고·보조 자료)

**포함 범위:**

- Deepfake **+** Photoshop·편집·스plicing 등 **위변조** (이미지 계획서 5모듈과 정합)
- **둘 다** 가져감 — 점수 모두 높으면 “합성 + 이후 변조” 등 **복합 결론**

### 5.2 사용자 플로우 (재설계)

#### Flow A — **최초 등록 (증거 등록 + ①)**

```text
파일 업로드
  → (병렬) ① Deepfake/합성 분석 → ④ 구간별 의심 점수 + ⑤ XAI
  → SHA-256
  → 디지털 서명 (원본 manifest)
  → S3 original (Object Lock) + CoC Ledger
  → 상세 페이지 / PDF(로드맵)
```

#### Flow B — **비교 검증 (②)**

```text
[내 분석 기록] 등에서 기존 증거(서명된 원본) 선택
  → "비교" UI (신규 — 대시보드 업로드 UI와 분리)
  → 비교 대상 파일 업로드
  → 타겟: 복사본 생성 → 서명 → 서명된 원본과 비교 (원본 바이트 직접 분석 X)
  → ② 위변조 분석 + ④ 결과 + ⑤ XAI
  → 비교 결과 전용 상세 페이지
```

**GPU `/tmp`:** 분석용 **임시 복사본**만 — **S3 copy prefix 없음** (팀 확정)

### 5.3 디지털 서명 — 어디에?

| 시점 | 서명 대상 | 비고 |
|------|-----------|------|
| **최초 등록** | **원본 manifest** (해시 + evidenceId + 시각 + …) | S3 **원본 파일 자체**는 변경 없음; **서명은 메타/manifest** |
| **비교 시** | **타겟 파일의 복사본** manifest 서명 | 원본에 서명된 기준과 **서명된 끼리** 비교 |
| **원본** | 등록 시 **1회** 서명·저장 | 이후 비교 기준점 |

**멘토·팀 합의:** “서명하면 파일이 바뀐다” 혼선 → **원본 bytes는 그대로**, **서명은 별도 레코드/ manifest** 또는 **등록 시점 원본에 대한 서명 레코드**로 관리. 타겟은 **복사본 + 서명** 후 비교.

### 5.4 XAI (Explainable AI)

- **청크/구간별** 의심 점수 시각화 (영상: 타임라인·히트맵, 음성: 스펙트rogram·구간, 이미지: ELA heatmap)
- **① Deepfake** / **② 위변조** **각각** 해당 구간 표시
- PDF §3: riskScore + **근거 시각화** (Phase 2)

### 5.5 PDF 리포트 (목표 항목)

- AI 종합 위험 평가 · **탐지 근거** · **의심 구간(XAI)**
- 메타데이터 이상 · **CoC / 무결성 로그** 요약
- 법정 제출 **가능 형식** (완성은 MVP 이후)

## 6. Chain of Custody · 용어 정리 (발표용)

| 용어 | ForenShield에서의 의미 |
|------|------------------------|
| **SHA-256** | 파일 **내용 지문** |
| **디지털 서명** | 기관/시스템이 **“이 해시·manifest를 등록했다”** 공인 |
| **해시 체인 (CoC)** | `custody_logs` **previousLogHash → currentLogHash** (로그 **조작 탐지**) |
| **Ledger** | CoC + evidence 메타 **append-only** 기록 |
| **블록체인/TSA** | (로드맵) manifest 해시 **외부 타임스탬프** — **파일 저장 X** |
| **Object Lock** | S3 **원본 바이트** 삭제·덮어쓰기 방지 |

**“자동화된 무결성 인증 체인” 발표 문장 예:**

> “업로드 즉시 SHA-256 기준점을 만들고, 기관 키로 manifest에 디지털 서명하며, 모든 이벤트는 CoC 해시 체인에 기록됩니다. 원본 파일은 S3 Object Lock으로 보호하고, 검증 API로 재해시·체인·서명을 확인합니다.”

## 11. 발표·문서에 넣을 문장 (Copy-ready)

### 11.1 프로젝트 한 줄

> ForenShield는 **AI 기반 Deepfake·합성 의심 탐지**와 **등록 증거 대비 위·변조 비교 검증**을 제공하고, **SHA-256·디지털 서명·CoC 해시 체인·S3 Object Lock**으로 **증거 무결성**을 지원하는 포렌식 플랫폼입니다.

### 11.2 S3 vs 블록체인

> **증거 파일(Gb급)은 S3 Object Lock**에 보관하고, **블록체인(또는 TSA)에는 해시·서명 manifest만** 앵커하여 **외부 시각 증명**을 보강합니다. 블록체인은 **저장소가 아니라 무결성 레이어**입니다.

### 11.3 MVP 범위

> 발표 시점 MVP는 **증거 등록(해시·서명·S3) + 단일 모달 분석 + 구간 XAI + (가능 시) 비교 검증 UI**에 집중하며, C2PA·on-chain·전 모달 **풀 완성은 로드맵**입니다.
