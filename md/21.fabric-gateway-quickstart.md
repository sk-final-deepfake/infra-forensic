# Fabric Gateway — EC2 배포 가이드 (ForenShield)

> **풀 가이드:** [`22.fabric-ec2-full-runbook.md`](./22.fabric-ec2-full-runbook.md) (콘솔·CLI·**SSM 접속**)

---

## 아키텍처

```text
[EKS backend Pod]  ──VPC 내부──►  [EC2 Fabric :8088]  →  Fabric ledger
       │
       └── VPN ──► [On-Prem GPU :8000]   (AI만, 기존과 동일)
```

| 서비스 | 위치 | 포트 | ConfigMap |
|--------|------|------|-----------|
| AI 추론 | On-Prem GPU | 8000 | `AI_GATEWAY_URL` |
| 블록체인 앵커 | **EC2** | 8088 | `BLOCKCHAIN_ANCHOR_URL` |

GPU 서버에 Fabric을 올릴 **필요 없음**.

---

## 1. EC2 스펙 (PoC)

| 항목 | 권장 |
|------|------|
| 인스턴스 | `t3.medium` (2 vCPU / 4GB) — 부족하면 `t3.large` |
| OS | Ubuntu 22.04 LTS |
| VPC | **EKS 클러스터와 동일 VPC** |
| 서브넷 | **Private subnet** (EKS worker와 통신 가능) |
| Public IP | **없음** (SSM Session Manager로 접속) |
| EBS | 30GB+ (Docker + Fabric 이미지) |
| 이름 태그 | `forenshield-fabric-poc` |

---

## 2. Security Group

**새 SG:** `forenshield-sg-fabric`

| 방향 | 포트 | 소스 | 용도 |
|------|------|------|------|
| Inbound | **8088** | **`forenshield-sg-eks-node`** | BE → Gateway |
| Outbound | All | 0.0.0.0/0 | Docker pull, git clone (NAT 경유) |

> SSH 22는 열지 않음. EC2 접속은 **SSM** ([22번 문서](./22.fabric-ec2-full-runbook.md) Step A·D).

> 8088을 **0.0.0.0/0**에 열지 마세요. EKS worker SG만 허용.

**AWS CLI 예시** (변수는 환경에 맞게):

```bash
export AWS_PROFILE=forenshield
export AWS_REGION=ap-northeast-2
export VPC_ID=vpc-xxxx
export EKS_NODE_SG=sg-xxxx   # backend-ng 노드 SG

FABRIC_SG=$(aws ec2 create-security-group \
  --group-name forenshield-fabric-sg \
  --description "ForenShield Fabric Gateway" \
  --vpc-id "$VPC_ID" \
  --query GroupId --output text)

aws ec2 authorize-security-group-ingress \
  --group-id "$FABRIC_SG" \
  --protocol tcp --port 8088 \
  --source-group "$EKS_NODE_SG"
```

## 3. EC2 생성 후 접속 (SSM)

**콘솔:** EC2 → 인스턴스 → **연결** → **Session Manager** → 연결

**CLI:**

```bash
aws ssm start-session --target <INSTANCE_ID>
```

IAM 인스턴스 프로필 + `AmazonSSMManagedInstanceCore` 필요. 상세는 [22번](./22.fabric-ec2-full-runbook.md).

## 5. Fabric + chaincode 설치

```bash
cd ~/forenshield-infra/fabric
bash scripts/setup-all.sh
```

완료 시 `print-be-config.sh` 출력에 **EKS에 넣을 URL**이 나옵니다.

---

## 6. Gateway 실행

**포그라운드 (테스트):**

```bash
bash scripts/start-gateway.sh
```

**백그라운드:**

```bash
cd gateway && nohup npm start > gateway.log 2>&1 &
```

**재부팅 후 자동 (선택):**

```bash
# 경로·User 맞게 수정 후
sudo cp systemd/forenshield-fabric-gateway.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now forenshield-fabric-gateway
```

## 7. EC2에서 자체 확인

```bash
curl http://localhost:8088/health

curl -X POST http://localhost:8088/api/v1/anchor \
  -H 'Content-Type: application/json' \
  -d '{
    "subjectHash":"deadbeef",
    "anchorType":"EVIDENCE_HASH",
    "clientId":"forenshield-be",
    "evidenceId":"1"
  }'
```

`transactionHash` 수신 → Fabric OK.

---

## 8. EKS 백엔드 연결

EC2에서:

```bash
bash scripts/print-be-config.sh
```

출력 예:

```yaml
BLOCKCHAIN_ANCHOR_URL: "http://10.0.12.34:8088/api/v1/anchor"
```

[`config/k8s/app-config.yaml`](../config/k8s/app-config.yaml) 수정:

```yaml
BLOCKCHAIN_ANCHOR_ENABLED: "true"
BLOCKCHAIN_ANCHOR_MODE: "http"
BLOCKCHAIN_ANCHOR_URL: "http://10.0.12.34:8088/api/v1/anchor"   # EC2 private IP
BLOCKCHAIN_ANCHOR_NETWORK: "hyperledger-fabric-forenshield"
```

적용:

```bash
kubectl apply -f config/k8s/app-config.yaml -n forenshield
kubectl rollout restart deployment/backend -n forenshield
```

**EKS Pod → EC2 연결 테스트** (backend Pod 안에서):

```bash
kubectl exec -n forenshield deploy/backend -- \
  curl -s -o /dev/null -w "%{http_code}" http://10.0.12.34:8088/health
# 200 기대
```

E2E: 증거 업로드 → `blockchain_anchors.status = ANCHORED`

## 10. 트러블슈팅

| 증상 | 확인 |
|------|------|
| EKS → EC2 Connection refused | SG 8088, EC2 private IP, Gateway 프로세스 |
| `502 fabric_submit_failed` | `gateway.log`, `docker ps` (peer/orderer) |
| chaincode deploy 실패 | `setup-all.sh` 재실행, 디스크 30GB+ |
| AI는 되는데 블록체인만 실패 | AI=VPN On-Prem, Fabric=EC2 — **URL 혼동** 확인 |
