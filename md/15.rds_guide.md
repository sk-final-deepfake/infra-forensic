## 0. 사전 준비

로컬 PC **PowerShell**에서 한 번 실행합니다.

```powershell
$env:AWS_PROFILE = "forenshield"
$env:AWS_REGION = "ap-northeast-2"
aws eks update-kubeconfig --name forenshield --region ap-northeast-2
```

Git Bash를 쓰는 경우:

```bash
export AWS_PROFILE=forenshield
export AWS_REGION=ap-northeast-2
aws eks update-kubeconfig --name forenshield --region ap-northeast-2
```

## 1. DB 접속 정보 확인 (K8s Secret)

아래 블록을 **한 번에 복사**해 PowerShell에 붙여넣으세요.

```powershell
kubectl get secret db-credentials -n forenshield -o json | ConvertFrom-Json | ForEach-Object {
  $_.data.PSObject.Properties | ForEach-Object {
    $name = $_.Name
    $val = [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($_.Value))
    Write-Host "$name = $val"
  }
}
```

### 접속 정보 요약

| 항목 | 값 |
|------|-----|
| Host | `forenshield-db.chcswakki5dc.ap-northeast-2.rds.amazonaws.com` |
| Port | `5432` |
| DB | `forenshield` |
| User | `forenshield` |
| Password | 위 출력의 `POSTGRES_PASSWORD` |

> **Tip:** 비밀번호에 `$`가 있으면 반드시 **작은따옴표(`'`)** 로 감싸세요.  
> PowerShell/Git Bash에서 `$`는 변수로 해석될 수 있습니다.

```bash
export PGPASSWORD='admin123$'   # OK
export PGPASSWORD=admin123$     # ❌ $ 뒤가 잘릴 수 있음
```

## 2. RDS 접속 (kubectl Pod)

RDS는 VPC 내부(Private)이므로 로컬 PC에서 직접 `psql`이 안 될 수 있습니다.  
EKS에서 **임시 psql Pod**를 띄워 접속합니다.

### 2.1 psql 클라이언트 Pod 실행

**터미널 1 (로컬):**

```bash
kubectl run psql-client --rm -it --restart=Never -n forenshield --image=postgres:16-alpine -- bash
```

### 2.2 Pod 안에서 RDS 접속

**Pod 내부:**

```bash
export PGPASSWORD='여기에_POSTGRES_PASSWORD'

psql -h forenshield-db.chcswakki5dc.ap-northeast-2.rds.amazonaws.com \
     -U forenshield \
     -d forenshield
```

접속 성공 시 프롬프트:

```text
forenshield=>
```

## 3. DB 조회 (psql)

PostgreSQL에서 자주 쓰는 명령어입니다. (MySQL `SHOW TABLES` 대신 `\dt` 사용)

```sql
-- 연결 확인
SELECT current_database(), current_user, version();

-- 테이블 목록 (MySQL SHOW TABLES 대신)
\dt

-- 데이터베이스 목록 (SHOW DATABASES; 대신)
\l

-- users 테이블 구조
\d users

-- 기존 계정 확인
SELECT user_id, login_id, email, name, role, status, deleted_at, created_at
FROM users
ORDER BY user_id;

-- psql 종료
\q
```

## 4. 로그인 조건 (백엔드 기준)

관리자로 로그인하려면 `users` 테이블 행이 아래 조건을 만족해야 합니다.

| 컬럼 | 필수 값 |
|------|---------|
| `role` | `ROLE_ADMIN` |
| `status` | `APPROVED` |
| `deleted_at` | `NULL` |
| `password` | BCrypt 해시 (평문 저장 불가) |

## 5. 비밀번호 BCrypt 해시 만들기

DB에는 **해시만** 저장합니다. 로그인할 때는 **평문 비밀번호**를 입력합니다.

### 방법 A — psql Pod와 별도 (권장)

**새 터미널 (로컬)** 에서 한 번 실행. `Admin123!`을 원하는 비밀번호로 바꿉니다.

```bash
kubectl run bcrypt-gen --rm -it --restart=Never -n forenshield --image=python:3-alpine -- \
  sh -c "pip install -q bcrypt && python -c \"import bcrypt; print(bcrypt.hashpw(b'Admin123!', bcrypt.gensalt(10)).decode())\""
```

출력 예:

```text
$2b$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

→ 이 값을 **복사**해 둡니다.

### 방법 B — SSM 노드 (Amazon Linux)

```bash
sudo dnf install -y httpd-tools
htpasswd -bnBC 10 "" 'Admin123!' | tr -d ':\n'
```

## 6. 관리자 계정 INSERT

**psql (`forenshield=>`)** 환경에서 실행합니다.  
5번에서 복사한 BCrypt 해시를 아래 `password` 값에 붙여넣으세요.

```sql
INSERT INTO users (
  login_id,
  email,
  password,
  name,
  phone,
  organization_type,
  department,
  position,
  role,
  status,
  dark_mode,
  created_at,
  updated_at
) VALUES (
  'admin01',
  'admin01@forenshield.local',
  '$2b$10$여기에_복사한_BCRYPT_해시',
  '시스템관리자',
  '010-0000-0000',
  'ETC',
  '운영팀',
  '관리자',
  'ROLE_ADMIN',
  'APPROVED',
  false,
  NOW(),
  NOW()
);
```

### 등록 확인

```sql
SELECT user_id, login_id, name, role, status, deleted_at
FROM users
WHERE login_id = 'admin01';
```

### 비밀번호만 재설정

```sql
UPDATE users
SET password = '$2b$10$여기에_새_BCRYPT_해시',
    updated_at = NOW()
WHERE login_id = 'admin01'
  AND deleted_at IS NULL;
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `psql: connection timed out` | 로컬에서 RDS Private IP 직접 접속 | §2 kubectl Pod 사용 |
| `password authentication failed` | `PGPASSWORD` 오타 · `$` 해석 | 작은따옴표로 감싸기 |
| 로그인 401/403 | role/status/deleted_at 불일치 | §4 조건 확인 |
| BCrypt 로그인 실패 | 해시 잘림·다른 비밀번호로 생성 | §5 다시 실행 후 UPDATE |
