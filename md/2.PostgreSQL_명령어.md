## 1. MySQL vs PostgreSQL


| 하려는 일  | MySQL ❌            | PostgreSQL ✅                               |
| ------ | ------------------ | ------------------------------------------ |
| DB 목록  | `SHOW DATABASES;`  | `\l` 또는 `SELECT datname FROM pg_database;` |
| 테이블 목록 | `SHOW TABLES;`     | `\dt` 또는 `information_schema` 조회           |
| DB 선택  | `USE forenshield;` | `\c forenshield`                           |
| 컬럼 구조  | `DESCRIBE users;`  | `\d users`                                 |
| 테이블 생성 | `CREATE TABLE ...` | 동일 (타입·자동증가 문법 일부 다름)                      |


### `SHOW databases` / `SHOW tables` 에러가 나는 이유

PostgreSQL의 `SHOW`는 **서버 설정 파라미터**만 조회합니다 (`SHOW timezone;` 등).

```text
ERROR: unrecognized configuration parameter "databases"
ERROR: unrecognized configuration parameter "tables"
```

→ MySQL 문법이 아니라 `**\l`, `\dt`, `\d**` 또는 **SQL(`information_schema`)** 을 사용합니다.

## 3. psql 메타 명령 (`\`)

`\`로 시작하는 명령은 **psql 전용**이며, SQL이 아닙니다. **세미콜론(`;`) 불필요.**

### 연결 · DB


| 명령               | 설명                      |
| ---------------- | ----------------------- |
| `\conninfo`      | 현재 접속 정보 (호스트, DB, 사용자) |
| `\c forenshield` | DB 변경 (MySQL `USE`)     |
| `\c postgres`    | 다른 DB로 전환               |
| `\l`             | 데이터베이스 목록               |
| `\l+`            | DB 목록 + 크기 등 상세         |


### 테이블 · 스키마


| 명령          | 설명                  |
| ----------- | ------------------- |
| `\dt`       | `public` 스키마 테이블 목록 |
| `\dt *.`*   | 모든 스키마 테이블          |
| `\dt+`      | 테이블 + 크기            |
| `\d users`  | 테이블 구조 (컬럼, PK, FK) |
| `\d+ users` | 상세 + 코멘트            |
| `\dn`       | 스키마 목록              |
| `\df`       | 함수 목록               |
| `\dv`       | 뷰 목록                |
| `\di`       | 인덱스 목록              |


### 사용자 · 권한


| 명령         | 설명         |
| ---------- | ---------- |
| `\du`      | 역할(사용자) 목록 |
| `\dp`      | 테이블 권한     |
| `\z users` | 특정 테이블 권한  |


### 기타


| 명령              | 설명                 |
| --------------- | ------------------ |
| `\?`            | psql 도움말           |
| `\h SELECT`     | SQL 문법 도움말         |
| `\timing on`    | 쿼리 실행 시간 표시        |
| `\x on`         | 세로 출력 (컬럼 많을 때)    |
| `\copy`         | 파일 ↔ 테이블 (클라이언트 측) |
| `\i script.sql` | SQL 파일 실행          |
| `\e`            | 에디터로 쿼리 작성         |

## 4. SQL 조회

GUI(DBeaver 등)나 스크립트에서 쓸 때 — **MySQL `SHOW` 대체.**

### 기본 확인

```sql
SELECT current_database(), current_user, version();
SELECT 1;   -- 연결 테스트
```

### DB · 테이블 · 컬럼

```sql
-- DB 목록
SELECT datname FROM pg_database ORDER BY datname;

-- public 스키마 테이블
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
  AND table_type = 'BASE TABLE'
ORDER BY table_name;

-- 컬럼 정보
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_schema = 'public'
  AND table_name = 'users'
ORDER BY ordinal_position;
```

### 데이터 조회

```sql
SELECT * FROM users LIMIT 10;
SELECT COUNT(*) FROM cases;
```

## 5. DDL · DML

### 테이블 생성 예시

```sql
CREATE TABLE users (
  id         BIGSERIAL PRIMARY KEY,
  email      VARCHAR(255) NOT NULL UNIQUE,
  name       VARCHAR(100),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_users_email ON users (email);
```

### MySQL과 자주 다른 점


| 항목       | MySQL            | PostgreSQL                                           |
| -------- | ---------------- | ---------------------------------------------------- |
| 자동 증가 PK | `AUTO_INCREMENT` | `SERIAL` / `BIGSERIAL` / `GENERATED ... AS IDENTITY` |
| 날짜·시간    | `DATETIME`       | `TIMESTAMP` / `TIMESTAMPTZ`                          |
| 식별자 따옴표  | ``column``       | `"column"` (대소문자 유지)                                 |


### INSERT · UPDATE · DELETE

```sql
INSERT INTO users (email, name) VALUES ('a@b.com', 'test');
UPDATE users SET name = 'changed' WHERE id = 1;
DELETE FROM users WHERE id = 1;
```

### 트랜잭션

```sql
BEGIN;
-- ... DML ...
COMMIT;     -- 취소: ROLLBACK;
```

### 테이블 삭제

```sql
DROP TABLE IF EXISTS users CASCADE;
```

## 6. `SHOW` — PostgreSQL에서 가능한 것

```sql
SHOW server_version;
SHOW timezone;
SHOW search_path;
SHOW ALL;    -- 주요 설정 일괄
```


| 명령                | 가능 여부     |
| ----------------- | --------- |
| `SHOW timezone;`  | ✅         |
| `SHOW databases;` | ❌ → `\l`  |
| `SHOW tables;`    | ❌ → `\dt` |

## 7. RDS 운영 확인

```sql
-- 활성 연결 수
SELECT count(*) FROM pg_stat_activity;

-- DB 크기
SELECT pg_size_pretty(pg_database_size('forenshield'));

-- 테이블별 크기 (테이블 생성 후)
SELECT relname,
       pg_size_pretty(pg_total_relation_size(relid)) AS total_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC;
```

### `\dt`가 비어 있을 때

인프라만 구축한 상태면 **테이블이 없는 것이 정상**입니다.  
Spring Boot + Flyway/Liquibase/JPA 마이그레이션 후 `\dt`에 테이블이 생깁니다.

## 9. 한 줄 치트시트

```text
\l          DB 목록
\c dbname   DB 전환
\dt         테이블 목록
\d table    테이블 구조
\du         사용자(역할)
\q          종료
```

**기억하기:** MySQL `SHOW ...` → PostgreSQL `**\dt` / `\l` / `\d`**
